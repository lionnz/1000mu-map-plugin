# 变更日志

本文件记录 1000MU 3D MAP 插件的所有版本变更。

---

## [v0.3.17] - 2026-07-15

### 面板布局调整
- **着色方式移入导入页**：「视图着色检查」从导出-实用小工具移入导入-视图设置板块，重命名为「着色方式」并增加图标
- **清理工具移入导出页**：「从该文件中清理未使用的数据」从导入页移入导出-实用小工具首位，与修复工具归类统一

### 内置高度预设调整
- **移除「文字」预设**：不再把"文字"作为独立关键词匹配，避免含"文字"的图层名（如"文字_支路名称"）被意外匹配为 0.5m 高度
- **新增路名专用预设**：`路名_主路` 20.10m、`路名_支路` 15.10m，放在"主路""支路"之后、保证含"路名_XX"的图层名先匹配到精确的路名预设
- **预设表重新排序**：精确关键词优先（路名_XX → 商场边框 → 商场），避免子串误匹配（如「商场」误匹配「商场边框」）。其他按视觉层级排列：BOX系列 → 道路 → 绿化 → 水
- **新增预设自动升级机制**：通过 `BUILTIN_PRESETS_HASH` 指纹检测内置预设变更，插件启用时自动同步到偏好设置，解决更新插件后偏好设置仍显示旧预设的问题

### BUG 修复
- **修复清理提示红叉BUG**：根因是 UI 中使用了无效图标 `ROCKET`（Blender 4.2 不存在），导致 `draw()` 每次重绘抛出 `TypeError`，污染报告系统使后续 operator 的 INFO 提示显示为红叉。改为有效图标 `PLAY`
- **清理实现优化**：`purge_scene` 改为手动 `bpy.data.X.remove()`，避免 `orphans_purge` 内层 operator 的 ERROR 报告穿透到状态栏
- **修复 SVG 嵌套 `<g>` 导致错误的图层命名**：`apply_target_id` 递归遍历 SVG 时，嵌套子 `<g>` 的 id 会覆盖父组 id，导致出现不应存在的图层（如「路」出现在「支路」内、「建筑」出现在「浅色box」内）。修复方案：(a) 新增 `current_target_id is None` 守卫，已设置父组名后不允许嵌套 `<g>` 覆盖；(b) 在调用处检测 SVG 根节点下唯一的 `<g>` 包装层（Figma 导出的 Frame），从其子元素开始递归而非从 Frame 本身开始
- **修复 SVG 颜色解析的嵌套 `<g>` 覆盖**：`parse_svg_colors` 的 `traverse()` 存在与 `apply_target_id` 相同的嵌套 `<g>` 覆盖 bug，导致颜色被关联到错误的图层名。同步修复
- **移除导入页残留的「一键挤出」按钮**：导入页底部曾有一个与挤出页共享 `layer_list` 显示条件的「一键挤出」按钮，导致在挤出页点击「刷新图层列表」后、切回导入页时会在底部出现一个孤立的挤出按钮。该按钮已删除，"一键挤出"仅在挤出页提供
- **新增 `context.screen` / `context.area` 守卫**：`view_tools.py` 和 `repair_tools.py` 中直接访问 `context.screen.areas` 和 `context.area.regions`，在无屏幕或无区域的上下文中会抛 `AttributeError`。已添加 `if context.screen:` / `if context.area:` 提前返回的空值守卫

### 功能移除
- **移除「对所选曲线物体进行网格优化」功能**：`MAP_OT_optimize_curve` 操作符及挤出-网格平面优化中的对应按钮已完全移除，该功能在实际使用中较鸡肋

### 提示文案优化
- 0结果时给出「无需处理」提示（如「场景很干净，没有孤立数据块」）
- 有失败时用 WARNING 级别报告，全成功时简洁明了

### 代码分块调整
- `build_atlas` 和 `purge_scene` 移入 `repair_tools.py`，删除 `scene_tools.py`
- 代码分块与 UI 面板对应：pipeline.py（核心流程）、view_tools.py（视图设置）、mesh_tools.py（网格优化）、repair_tools.py（实用小工具）

### 调试经验沉淀
- 新增 [blender-plugin-debug-guide.md](docs/blender-plugin-debug-guide.md)，记录终端启动 Blender 定位 GUI BUG 的方法

## [v0.3.16] - 2026-07-10

### 重新拓扑优化
- **新增前置优化步骤**：提取轮廓线前先执行「按距离合并（0.0001m）+ 有限融并（0.1°）」，清理 GLB 三角化后产生的重合顶点和共线折点，显著提升退化网格的重新拓扑成功率
- **新增末尾面稳定化**：烘焙成网格后执行「面三角化（BEAUTY）→ 三角面转四边面」，用 Blender 原生算法重建稳定拓扑，降低 GLB 导出三角化时产生0面积面的概率

### 导入网格优化
- **导入即执行面稳定化**：FillCurve 烘焙成网格后同样执行「面三角化 → 三角面转四边面」，从源头保证导入网格的拓扑稳定性

### 0面积面检查改进
- **归入改为移动至集合**：问题物体 link 到「0面积的面」集合后，从原属集合（含场景集合）unlink，避免一物体同时出现在多个集合导致大纲视图混乱

### 视图设置扩展
- **新增×1000预设**：Clip Start 10m / End 1000000m，适配超大比例地图。视图裁剪预设由 5 档扩展为 6 档

## [v0.3.15] - 2026-07-09

### 重大变更
- **插件更名**：「1000MU3D周边地图插件」→「1000MU 3D MAP 插件」
- **源码模块化拆分**：894 行单文件 `__init__.py` 拆分为 9 个模块（constants / utils / props / operators/ / ui / preferences）
- **图层列表动态生成**：从"导入时一次性填充"改为"基于当前场景选中物体动态生成"（移除 `is_active` 眼睛列）
- **导出改用原生弹窗**：移除路径/文件名输入框，点击导出GLB打开 Blender 原生 glTF 导出弹窗（移除 `export_dir` / `export_filename`）
- **标签页更名**：「配置」→「挤出」

### 导入页新增
- 视图设置：5 档 Clip Start/End 一键切换（默认/远点×10/×10/远点×100/×100）
- 场景清理：递归 purge 未使用数据（本地+已关联+递归）
- 导入方案B优化：SVG 转网格前插入几何节点填充曲线（N-gons 模式），从源头避免0面积面
- 导入页去 box 边框改扁平表单

### 挤出页新增
- 网格平面优化模块：
  - 0面积面检查：面积 < 0.0001 的面归类到集合「0面积的面」并报告
  - 重新拓扑（方案A）：8步自动化（删松散→轮廓线→曲线→几何节点填充→网格），通用兜底
- 刷新图层列表按钮：基于选中网格按 base_name 去重生成
- 一键挤出改为仅处理选中物体
- 图层列表列顺序：颜色→图层名→随机→高度（移除眼睛列）

### 导出页新增
- 实用小工具模块：
  - UVmap 批量清理：归一为单一 UVMap 层
  - 自定义法向批量清理：合并自独立插件，modal+timer 分批处理
  - 按所选面批量设原点：编辑模式多物体批量，完整面坐标系对齐
  - 材质球打包、生成色块贴图（从原模块移入）
  - 视图着色三按钮：默认着色 / paint.sl+材质 / paint.sl+纹理
- 复选框改全宽 Toggle 纵排，文案加长
- 拼音防呆：方案A（改名+计时器还原），待验证方案B（glTF扩展）

## [v0.3.14] - 2026-07-08

### 重大变更
- **内置拼音引擎**：将 pypinyin 打包进插件（vendor），拼音防呆功能开箱即用，不再需要联网安装
- **移除 pip 安装机制**：删除 `MAP_OT_install_env` 操作符及导出页的拼音安装警告 UI
- **安装方式变更**：从单 .py 文件改为 .zip 包（因需内置 pypinyin 数据文件）

### 融合优化（v0.3.9 – v0.3.13）
- 兼容 Blender 4.5：`material.shadow_method` 改为 `hasattr` 兼容检测
- 偏好设置可配置高度预设表
- 色块贴图：支持单物体多材质球 + 冗余槽自动合并 + 回归修复
- 法线统一隔离、非递归孤立清理、epsilon 防抖、md5 防重名、tempfile 隔离
- 随机高度开启时自动填充区间、UI 布局优化、description 悬浮提示
- 新增 `_on_use_rand_height` 回调

## [v0.3.8] - 2026-07-07

### 初始版本
- 用户使用 AI coding 完成的初版插件。
- 核心功能：SVG 导入、物理换算、图层预设、挤出、基础材质生成、双轨制图集、GLB 导出。

### 已知问题（均已在 v0.3.14 中修复）
- 单物体多材质球时只贴首个材质
- 挤出末尾法线统一会误伤场景其他物体
- 递归孤立数据清理会误删用户保留资产
- 多 Blender 实例同时导入 SVG 会冲突
- 拼音功能需联网安装 pypinyin
