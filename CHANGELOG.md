# 变更日志

本文件记录 1000MU 3D MAP 插件的所有版本变更。

---

## [v0.3.15] - 2026-07-09

### 重大变更
- **插件更名**：「1000MU3D周边地图插件」→「1000MU 3D MAP 插件」
- **源码模块化拆分**：894 行单文件 `__init__.py` 拆分为 9 个模块（constants / utils / props / operators/ / ui / preferences）
- **图层列表动态生成**：从"导入时一次性填充"改为"基于当前场景选中物体动态生成"（移除 `is_active` 眼睛列）
- **导出改用原生弹窗**：移除路径/文件名输入框，点击导出GLB打开 Blender 原生 glTF 导出弹窗（移除 `export_dir` / `export_filename`）
- **标签页更名**：「配置」→「挤出」

### 导入页新增
- 视图设置：5 档 Clip Start/End 一键切换（默认/远点×10/×10/远点×100/×100）
- 场景清理：递归 purge 未使用数据（本地+已关联+递归）
- 导入方案B优化：SVG 转网格前插入几何节点填充曲线（N-gons 模式），从源头避免0面积面
- 导入页去 box 边框改扁平表单

### 挤出页新增
- 网格平面优化模块：
  - 0面积面检查：面积 < 0.0001 的面归类到集合「0面积的面」并报告
  - 重新拓扑（方案A）：8步自动化（删松散→轮廓线→曲线→几何节点填充→网格），通用兜底
- 刷新图层列表按钮：基于选中网格按 base_name 去重生成
- 一键挤出改为仅处理选中物体
- 图层列表列顺序：颜色→图层名→随机→高度（移除眼睛列）

### 导出页新增
- 实用小工具模块：
  - UVmap 批量清理：归一为单一 UVMap 层
  - 自定义法向批量清理：合并自独立插件，modal+timer 分批处理
  - 按所选面批量设原点：编辑模式多物体批量，完整面坐标系对齐
  - 材质球打包、生成色块贴图（从原模块移入）
  - 视图着色三按钮：默认着色 / paint.sl+材质 / paint.sl+纹理
- 复选框改全宽 Toggle 纵排，文案加长
- 拼音防呆：方案A（改名+计时器还原），待验证方案B（glTF扩展）

---

## [v0.3.14] - 2026-07-08

### 重大变更
- **内置拼音引擎**：将 pypinyin 打包进插件（vendor），拼音防呆功能开箱即用，不再需要联网安装
- **移除 pip 安装机制**：删除 `MAP_OT_install_env` 操作符及导出页的拼音安装警告 UI
- **安装方式变更**：从单 .py 文件改为 .zip 包（因需内置 pypinyin 数据文件）

### 融合优化（v0.3.9 – v0.3.13）
- 兼容 Blender 4.5：`material.shadow_method` 改为 `hasattr` 兼容检测
- 偏好设置可配置高度预设表
- 色块贴图：支持单物体多材质球 + 冗余槽自动合并 + 回归修复
- 法线统一隔离、非递归孤立清理、epsilon 防抖、md5 防重名、tempfile 隔离
- 随机高度开启时自动填充区间、UI 布局优化、description 悬浮提示
- 新增 `_on_use_rand_height` 回调

---

## [v0.3.8] - 2026-07-07

### 初始版本
- 用户使用 AI coding 完成的初版插件。
- 核心功能：SVG 导入、物理换算、图层预设、挤出、基础材质生成、双轨制图集、GLB 导出。

### 已知问题（均已在 v0.3.14 中修复）
- 单物体多材质球时只贴首个材质
- 挤出末尾法线统一会误伤场景其他物体
- 递归孤立数据清理会误删用户保留资产
- 多 Blender 实例同时导入 SVG 会冲突
- 拼音功能需联网安装 pypinyin
