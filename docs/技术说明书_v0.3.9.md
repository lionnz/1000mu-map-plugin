# 1000MU 3D 周边地图插件 (Figma to Blender) 说明书

- **插件名称**：1000MU 3D周边地图插件 Figma to Blender
- **当前版本**：v0.3.9
- **适用环境**：Blender 4.2.0 及以上
- **入口位置**：View3D > 侧边栏 (N 面板) > `1000Map` 标签
- **维护者**：1000MU 设计团队内部工具
- **定位**：商业地产导视系统 3D 资产生产管线，实现从 Figma 2D 矢量到 WebGL 轻量化 3D 资产的高自动化交付。

---

## 版本变更摘要（v0.3.8 → v0.3.9）

| # | 变更 | 说明 |
|---|---|---|
| 1 | **多材质球物体支持** | 色块贴图引擎由「按物体聚合」改为「按材质聚合 + 按面写 UV」，单物体含多个材质球时无需分离即可正确贴图，零顶点增量 |
| 2 | 法线统一隔离 | 挤出末尾的 `normals_make_consistent` 仅作用于地图物体，不再误伤场景其他资产 |
| 3 | 孤立数据清理降级 | `orphans_purge` 改为非递归，避免误删用户暂时未引用但希望保留的资产 |
| 4 | 法线翻转 epsilon | `force_normals_up` 引入 `epsilon=1e-5`，防止浮点抖动导致的翻转误判 |
| 5 | 拼音降级防重名 | 纯中文名无 ASCII 字符时改用 md5 哈希前缀（`layer_<8位>`），避免全部重名为 `layer` |
| 6 | 临时文件隔离 | SVG 导入改用 `tempfile.mkstemp`，避免无写权限或多 Blender 实例同时导入冲突 |
| 7 | 异常可见 | 关键路径减少裸 `except: pass`，打印 `[1000Map]` 前缀错误信息便于排查 |

## 一、功能概览

本插件围绕「**导入 → 预设 → 挤出 → 优化 → 导出**」一条主线，将街区沙盘制作中重复的几何处理、单位换算、UV 与材质整理等工作自动化。

### 1. 自动化范围

- SVG 数据清洗与 Figma 中文层名逆向解码
- 像素到物理米制的尺度换算与全局居中
- 语义化挤出（按图层关键词匹配默认高度）
- 随机高度错落（可选区间）
- 法线强制朝上、Solidify 挤出、Weld 焊接、Triangulate 三角化
- 基础材质（Principled BSDF）自动生成，支持透明通道
- 全局色块图集（Color Atlas）+ 双轨制 Opaque/Transparent 材质分流
- **单物体多材质球支持（v0.3.9 新增）**：按面写入 UV，无需分离物体
- 显存级 UV 坍缩（`foreach_set` 批量写入）
- GLB 导出（含中文→拼音防呆、可见性/修改器应用选项）

### 2. 人工边界

- SVG 中若存在「0 面积面 / 共线顶点」，挤出异常时请使用「有限融并 (Limited Dissolve)」修复拓扑。
- pypinyin 库未安装时进入「字符剔除降级模式」，导出文件名仅保留 ASCII 字符；如需保留中文语义的拼音命名，请先执行「初始化/修复拼音环境」。
- 插件仅处理已注册到图层列表中的网格物体，不会影响场景内其他资产。
- 孤立数据清理为**非递归**，仅清理顶层未引用数据；如需深度清理，请手动在大纲层执行递归清理。

## 二、安装指南

1. 打开 Blender（推荐 **4.2 或更高版本**）。
2. 顶部菜单 **编辑 (Edit) → 偏好设置 (Preferences)**。
3. 左侧选择 **插件 (Add-ons)**。
4. 右上角点击 **从磁盘安装 (Install from Disk)**，选择 `1000MU_Map_Plugin_v039.py`。
5. 安装后 Blender 会自动勾选启用插件；在偏好设置左下角点击 **保存用户设置**，或开启 **自动保存设置** 以持久化。
6. 回到 3D 主视图，按 `N` 键打开侧边栏，点击 **`1000Map`** 标签进入工作流。

> 首次使用如需「拼音防呆」功能保留中文语义，请在「导出」标签页点击「初始化/修复拼音环境」（需联网，后台安装 pypinyin）。

> 升级提示：如已安装 v0.3.8，请先在插件管理中禁用并卸载旧版，再安装 v0.3.9。两个版本共用同一 `1000Map` 面板与 `map.*` 操作 ID，不可并存。

## 三、标准工作流

插件侧边栏采用 **三标签页** 结构：`导入 / 预设 / 导出`，并会在关键操作后自动切换到下一步所需标签页。

### 标签页 1：📦 数据导入

| 控件 | 说明 |
|---|---|
| SVG 文件 | 选择 Figma 导出的 `.svg` 文件 |
| `px = m` 比例 | 团队标准默认 `10px = 1m`，可按项目调整 |
| 曲线精度 | 控制曲线 `resolution_u`，1–12，默认 4；越高越平滑、面数越多 |
| 导入并重构空间 | 解码层名 → 烙印层级 → 物理换算 → 居中 → 转网格 → 提取颜色 → 生成图层列表 |

**执行后行为**：
- 自动切换到「预设」标签页。
- 顶部提示 `✅ 已导入 N 个图层`，并出现「一键挤出」入口。

### 标签页 2：🏗️ 预设挤出

**图层列表（UIList）每行字段**：

| 字段 | 控件 | 说明 |
|---|---|---|
| 启用 `is_active` | 眼睛图标 | 关闭后该图层不参与挤出 |
| 图层名 `layer_name` | 文本（只读） | 已剥离 `.001` 等系统后缀的业务基准名 |
| 随机高度 `use_rand_height` | SHADERFX 图标 | 开启后显示 `最小 ~ 最大` 区间输入 |
| 高度 `height` | 数值输入 | 随机关闭时使用，单位 m |
| 颜色 `color` | 色板 | 从 SVG 提取的 16 进制色（已转线性），可手动覆盖 |

**底部「一键挤出」按钮**（子面板，标签页 1 完成导入后也会出现入口）：
- 仅处理图层列表中注册的网格，清空其原有修改器。
- 对每个物体：法线 Z-Lock → Solidify（厚度=高度）→ Weld → Triangulate → 生成/复用 `Mat_<图层名>` 材质。
- 高度为 0 时回退为 1–10m 随机值，保证始终挤出。
- **法线统一仅作用于地图物体**（v0.3.9 修复），不会影响场景中其他资产。
- 最后执行**非递归**孤立数据清理。
- 完成后自动切换到「导出」标签页。

### 标签页 3：📤 导出交付

**🎨 渲染与性能优化**
- **生成色块贴图**：双轨制图集引擎，**按材质聚合**（v0.3.9 重构），支持单物体多材质球，生成 `3DMap_ColorAtlas` 贴图与 `Mat_3DMap_Atlas_Opaque` / `Mat_3DMap_Atlas_Transparent` 两套材质，UV 按面坍缩指向色块中心。

**📤 导出交付**
- 路径 / 文件名（自动补 `.glb` 后缀）
- 三个开关：
  - `仅导出可见`：跳过 `hide_viewport` 物体
  - `应用修改器`：导出时烘焙修改器栈
  - `拼音防呆`：导出前将中文物体/材质名转为拼音/ASCII，导出后**自动还原**原名
- 当检测到 pypinyin 未安装时，额外显示警告框与「初始化/修复拼音环境」按钮。

## 四、技术原理深度解析

下列各项均与 v0.3.9 源码行为一一对应。

### 4.1 Figma 字符解码引擎
Figma 导出的 SVG 常把中文层名转码为 `_x5546__x573a_` 之类。`decode_figma_id` 依次执行：`latin1→utf-8` 解码 → 正则替换 `_xXXXX_` 十六进制段 → `urllib.parse.unquote` URL 解码 → 去除 `_数字` 尾缀，还原业务命名。

### 4.2 XML 标签烙印（层级扁平化）
SVG 的 `<g>` 编组在 Blender 导入后会被丢弃。`apply_target_id` 在导入前递归遍历：父级 `<g>` 的解码名被作为 `current_target_id` 向下传递，所有子级形状（`path/rect/circle/...`）的 `id` 被强行改写为父级名，实现层级→扁平命名映射。`INVALID_LAYER_NAMES` 集合过滤掉 `svg/g/path/layer/group` 等无业务含义的标签名。

### 4.3 物理尺度绝对映射
固定乘数 `3543.0 × (ratio_m / ratio_px)` 作为 `scale_factor` 应用于所有曲线，随后 `Apply Scale` 冻结缩放为 1.0。默认 `10px=1m` 下，乘数为 354.3，确保沙盘具备物理真实尺寸。

### 4.4 视口居中算法
按 SVG 宽高与比例计算偏移 `offset_x = -(svg_w × ratio_m/ratio_px)/2`，对所有网格 `location.x/y` 施加偏移后 `Apply Location/Rotation`，强制几何中心落到原点 `(0,0,0)`。

### 4.5 viewBox 自适应捕获
`parse_dim` 优先读取 `width/height`（剥离 `px` 单位）；为空或解析失败时下穿读取 `viewBox` 的第 3、4 项；仍失败则兜底 `1000×1000`，避免百分比写法导致类型报错。

### 4.6 临时文件隔离（v0.3.9 改进）
导入时使用 `tempfile.mkstemp(suffix='.svg', prefix='blender_map_')` 生成系统临时文件，替代旧版固定的 `temp_blender_map.svg`。导入完成后立即删除。**收益**：避免源目录无写权限导致失败；避免多个 Blender 实例同时导入时互相覆盖临时文件。

### 4.7 智能层级聚合
`get_base_name` 用正则 `\.\d{3,}$` 剥离 Blender 的 `.001/.002` 系统后缀。导入时以 `seen_base_names` 去重，仅首次出现的基准名进入图层列表，避免冗余重名条目。

### 4.8 图层预设与色彩 Gamma 转换
- `DEFAULT_HEIGHT_CONFIG` 内置 10 类关键词默认高度（见附录 A）。
- 导入时若图层名包含关键词（如 `绿化`、`主路`），自动套用预设高度；未命中则 1–10m 随机。
- `parse_svg_colors` 提取每个形状的 `fill` 与 `fill-opacity/opacity`，`hex_to_float` 将 sRGB 16 进制经 `srgb_to_linear` 转换为 Blender 线性空间浮点色，避免 Gamma 异常。
- **同名图层只保留首次出现的颜色**（设计选择，避免颜色抖动）。

### 4.9 法线 Z-Lock 修复（v0.3.9 改进）
`force_normals_up(obj, epsilon=1e-5)` 通过 `bmesh` 在内存中遍历所有面，凡 `normal.z < -epsilon` 即翻转，再 `normal_update` 回写，确保建筑朝 +Z 生长。引入 `epsilon` 防止浮点抖动导致的翻转误判（如垂直面 `z≈0`）。挤出后还**仅对地图物体**全局执行一次 `mesh.normals_make_consistent(inside=False)` 做最终统一。

### 4.10 挤出修改器栈
每个物体依次附加：
1. `SOLIDIFY`（厚度=高度，`offset=1.0` 朝上挤出）
2. `WELD`（焊接重合顶点，清理拓扑）
3. `TRIANGULATE`（三角化，适配 WebGL）

> 高度回退策略：当配置高度 `≤ 0` 时，自动取 `random.randint(1,10)` 保证始终挤出，不产生 0 厚度平面。

### 4.11 法线统一隔离（v0.3.9 修复）
旧版在挤出末尾执行 `bpy.ops.object.select_all(action='SELECT')` 把场景所有物体一起进入编辑模式做 `normals_make_consistent`，会误伤反向法线刻意保留的资产。v0.3.9 改为仅选中 `is_map_obj` 判定通过的物体执行。

### 4.12 随机高度错落
图层级开关 `use_rand_height` 开启后，挤出时对该图层下每个物体在 `[rand_min, rand_max]` 区间独立取样（`random.uniform`，保留 2 位小数）。区间自动校正 `min/max` 顺序，`lo==hi` 时直接取该值。UI 中以 `最小 ~ 最大` 三段式输入呈现。

### 4.13 基础材质自动生成
按图层名为每个基准名生成 `Mat_<图层名>`（已存在则复用）：
- 节点图：Principled BSDF → Material Output
- `Base Color` / `Alpha` 取自图层色
- 不透明物：`Roughness=0.8`，`blend_method=OPAQUE`
- 透明物（Alpha<1）：`Roughness=0.2`，`blend_method=BLEND`，`shadow_method=NONE`
- `diffuse_color` 经 `linear_to_srgb` 回转，保证视口色与渲染一致

### 4.14 双轨制图集分流引擎（v0.3.9 重构：按材质聚合）

**旧版问题**：v0.3.8 按「物体」聚合，仅取 `obj.data.materials[0]`，导致单物体含多个材质球时只有第一个材质的颜色被贴图，其余材质的面全部错贴。手动「按材质分离」会增加大量顶点。

**v0.3.9 方案**：改为按「材质」聚合 + 按面写 UV，**零顶点增量**。

`MAP_OT_build_atlas` 流程：

1. **按材质聚合**：遍历所有可见网格的所有材质槽，以每个材质的 `(r,g,b,a)`（保留 3 位小数）为颜色键聚合。相同颜色的不同材质合并为同一色块。维护 `material_to_key`（材质→颜色键）与 `color_groups`（颜色键→材质集合）两张表。

2. **生成图集贴图**：网格列数 `ceil(sqrt(N))`，行数 `ceil(N/cols)`，图集尺寸取 `next_pow2` 的 2 的幂，单色块 `BLOCK=64` 像素。新建 `3DMap_ColorAtlas`（sRGB，alpha=True），按色块填充像素并 `pack()` 打包进 .blend。

3. **双轨材质生成**：
   - `Mat_3DMap_Atlas_Opaque`：`OPAQUE`，不接 Alpha，`Roughness=0.8`
   - `Mat_3DMap_Atlas_Transparent`：`BLEND` + `shadow_method=NONE`，接 Alpha，`Roughness=0.2`

4. **按面写入 UV**（关键改动）：
   - 对每个物体，预计算 `slot_uv`：`material_index → 色块 UV`。
   - 遍历 `me.polygons`，根据 `poly.material_index` 查找该面所用材质对应的色块 UV，给该面所有 loop 写入对应 UV。
   - 同一物体的不同面可指向不同色块，**无需分离物体**。
   - UV 经 `foreach_set` 批量坍缩到色块中心，并以 `±0.00005` 微偏防止 Mipmap 退化为 0（三角面/N-gon 均安全，不依赖四边面假设）。

5. **替换材质槽**：遍历每个物体的材质槽，按其颜色 Alpha 替换为 `mat_opaque` 或 `mat_trans`。**保持槽索引不变**，因此面的 `material_index` 仍然有效。同色多槽会指向同一个图集材质，存在冗余但不影响渲染。

**收益对比**：

| 维度 | v0.3.8（按物体） | v0.3.9（按材质+按面） |
|---|---|---|
| 多材质物体 | 只贴第一个材质 | 所有材质面正确贴图 |
| 是否需要分离物体 | 需要（增加顶点） | **不需要，零顶点增量** |
| 槽位变化 | 清空重建为 1 个 | 保留原槽数，仅替换槽内材质 |
| 面 `material_index` | 失效 | 保持有效 |
| UV 写入次数 | 物体数 × 1 | 物体数 × 1（仍按物体批量 `foreach_set`） |

### 4.15 显存级 UV 坍缩
使用 `uv_layer.data.foreach_set("uv", uv_flat)` 一次性写入全部 loop UV，绕过 Python 逐顶点循环，面对大型街区效率提升百倍。

### 4.16 拼音防呆与降级模式（v0.3.9 改进）
- `get_pypinyin()` 懒加载 pypinyin，不阻塞主线程。
- `chinese_to_safe_name`：库存在时转拼音下划线连接；不存在时进入降级模式。
- **v0.3.9 改进**：降级模式下，若 `fallback` 为空（纯中文名等无 ASCII 字符情况），改用 `layer_<md5前8位>` 而非统一的 `layer`，避免多个物体重名。
- 「初始化/修复拼音环境」按钮后台 `pip install pypinyin`（60s 超时）。
- 导出 GLB 时若开启 `拼音防呆`，**临时**改名 → 导出 → `finally` 块**还原原名**，保证 .blend 工程文件名不被破坏。还原失败会打印 `[1000Map]` 错误信息。

### 4.17 GLB 导出
调用 `export_scene.gltf`，关键参数：
- `export_format='GLB'`
- `use_visible=exp_visible_only`、`export_apply=exp_apply_modifiers`
- `export_materials='EXPORT'`，不含相机/灯光
- 导出目录不存在时自动 `makedirs`，文件名自动去除 `.glb` 重复后缀。

### 4.18 孤立数据清理（v0.3.9 降级）
挤出流程末尾执行 `bpy.ops.outliner.orphans_purge(do_local_ids=True, do_linked_ids=False, do_recursive=False)`，**非递归**清理顶层未引用数据。**收益**：避免误删用户暂时隐藏但希望保留的备用材质/网格。如需深度清理，请手动在大纲层执行递归清理。

### 4.19 标签页自动流转
`active_tab` 在关键操作后自动推进：导入成功 → 1（预设），挤出完成 → 2（导出），降低用户在面板间来回切换的认知成本。

## 五、设计团队制图规范

1. **关键词命名**：图层名包含业务词（如 `绿化`、`主路`、`浅色box`、`商场`、`水`），引擎自动匹配 `DEFAULT_HEIGHT_CONFIG` 默认高度。
2. **透明度设置**：水面、玻璃等组件在 Figma 中直接设置 `Fill Opacity`，插件读取 `fill-opacity/opacity` 并生成 `BLEND` 透明材质与透明图集轨道。
3. **几何拓扑**：建议在 Figma 内合并形状 (Union)，确保矢量闭合、无孤立点；遇 0 面积面可用「有限融并」修复。
4. **编组语义**：业务命名应放在 `<g>` 父级上，子级形状会继承父级名（4.2 烙印机制）。
5. **曲线精度**：圆弧/曲线较多的图稿可将「曲线精度」提高到 6–8；纯矩形图稿保持默认 4 即可，避免面数膨胀。
6. **多材质物体（v0.3.9）**：单个网格物体可包含多个材质球，色块贴图引擎会按面正确贴图，无需手动分离。若仍需按材质拆分为独立物体（如交付前端需要按材质分组），可手动「按材质分离」后再导出。

## 六、常见问题

| 现象 | 原因 / 处理 |
|---|---|
