# 1000MU 3D 周边地图插件 · 技术说明书 v0.3.14

> **适用版本**：v0.3.14 | **兼容 Blender**：4.2.0 – 4.5.x  
> **定位**：设计团队内部工具，Figma 2D 矢量地图 → WebGL 3D 资产全自动化生产管线  
> **安装方式**：ZIP 包（内置 pypinyin 拼音引擎，无需联网安装任何依赖）

---

## 一、插件简介

这个插件解决一个具体问题：**设计师在 Figma 里画好的 2D 商圈地图，怎么快速变成 3D 沙盘模型？**

传统做法是手动建模——一个一个地块挤出、调颜色、对齐尺寸——非常耗时。本插件把整个流程自动化了。

### 核心工作流

```
Figma 导出 SVG → 导入 Blender（自动换算比例）→ 配置高度和颜色 → 一键挤出 → 材质图集优化 → 导出 GLB
```

### v0.3.14 相比旧版的变化

| 变化 | 说明 |
|------|------|
| **内置拼音引擎** | pypinyin 打包进插件，不再需要联网安装，拼音防呆开箱即用 |
| **安装方式变更** | 从单 .py 文件改为 .zip 包（因为内置了 pypinyin 数据） |
| **兼容 Blender 4.5** | `shadow_method` 改为 `hasattr` 兼容检测 |
| **偏好设置可配预设** | 在 Blender 偏好设置里编辑高度预设表，所有项目共享 |
| **移除"初始化拼音环境"按钮** | 不再需要，拼音功能内置 |

## 二、安装

### 首次安装

1. 打开 Blender，**编辑 → 偏好设置 → 插件**
2. 点击右上角 **从磁盘安装**
3. 选择 `1000MU_Map_Plugin_v0314.zip`
4. 勾选启用，点击左下角 **保存用户设置**

5. 回到 3D 视图，按 `N` 键打开侧边栏，点击 **`1000Map`** 标签

### 升级（从旧版 v0.3.13 及更早）

旧版是单 `.py` 文件，v0.3.14 改为 `.zip` 包，必须彻底清除旧版残留才能正常安装：

1. **关闭 Blender**
2. 打开终端，执行清除命令：
   ```bash
   rm -rf ~/Library/Application\ Support/Blender/4.2/scripts/addons/1000MU_Map_Plugin*
   ```
3. 重新打开 Blender，进入 **编辑 → 偏好设置 → 插件**，检查列表里是否还有残留的 1000MU 条目（如有则点 **Remove** 彻底清理）
4. 再按上方「首次安装」步骤安装新版
5. 新旧版本不能同时启用

## 三、操作指南

插件面板有三个标签页：**导入 → 配置 → 导出**，按顺序操作。

### 标签 1：导入

| 步骤 | 操作 |
|------|------|
| **选 SVG** | 选择 Figma 导出的 `.svg` 文件。插件自动解码 Figma 中文层名 |
| **调比例** | 默认 `10px = 1m`，这是团队标准。可按项目调整 |
| **曲线精度** | 默认 4。圆弧多的图可以调到 6-8（更高=更平滑但面数更多） |
| **点导入** | 自动完成：解码层名 → 换算物理尺寸 → 全局居中 → 提取颜色 → 生成图层列表 |

> 导入后自动匹配高度预设表。未匹配的图层随机取 1-10m。

### 标签 2：配置

导入后会出现图层列表，每一行可以调整：

| 控件 | 作用 |
|------|------|
| 👁 眼睛图标 | 关掉则不参与挤出和导出 |
| 图层名 | 从 SVG 识别，只读 |
| 高度(m) | 修改挤出厚度 |
| 🎲 随机开关 | 开启后设一个区间，每个地块随机取高度，产生错落感 |
| 色块 | 从 SVG 取的填充色，可手动覆盖 |

配置好后点底部**「一键挤出」**。

**一键挤出做了什么**：
1. 翻转朝下的法线使其朝 +Z 生长
2. 按设定高度 Solidify 挤出
3. Weld 焊接顶点 + Triangulate 三角化（优化拓扑）
4. 自动生成 Principled BSDF 材质（透明物体自动 BLEND 模式）
5. 清理无用的孤立数据

> 插件只会处理注册到图层列表里的网格物体，不会动场景里其他东西。

### 标签 3：导出

分两步：

#### A. 生成色块贴图

点击 **「生成色块贴图」**，插件会：
- 把所有物体的材质颜色打包成一张 `3DMap_ColorAtlas` 贴图
- 自动分流不透明 / 透明两套图集材质
- 按面写 UV，一个物体多个材质球也正确处理（无需分离物体）
- 合并重复的材质槽，精简 GLB 导出列表

> 这一步必须在「一键挤出」之后执行。

#### B. 导出 GLB

| 选项 | 说明 |
|------|------|
| 路径 | 默认 `~/Downloads` |
| 文件名 | 自动补 `.glb` 后缀 |
| 仅导出可见 | 跳过隐藏的物体 |
| 应用修改器 | 导出时烘焙修改器效果 |
| 拼音防呆 | 中文名自动转拼音，防前端报错。导出后**自动还原原名**，不污染 .blend 文件 |

## 四、自定义高度预设

高度预设表可以在 Blender 偏好设置里编辑，不用改代码。

**位置**：编辑 → 偏好设置 → 插件 → 展开「1000MU 3D周边地图插件」

| 操作 | 方法 |
|------|------|
| 添加 | 点 `[+]` |
| 删除 | 选中一行点 `[-]` |
| 编辑 | 直接在表格里改关键词和高度 |
| 恢复默认 | 点「恢复内置默认值」 |

**匹配规则**：图层名**包含**关键词即命中。例如 `主路-辅道` 包含 `主路` → 自动设 20m。多个关键词同时命中时，按表格顺序优先匹配第一个。

### 内置默认值

| 关键词 | 高度 | 关键词 | 高度 |
|--------|------|--------|------|
| 绿化 | 10m | 深色box | 30m |
| 主路 | 20m | 文字 | 0.5m |
| 支路 | 15m | 商场边框 | 60m |
| 中色box | 40m | 商场 | 80m |
| 浅色box | 50m | 水 | 5m |

## 五、团队制图规范

为了插件能精准识别图层，建议在 Figma 中遵守以下约定：

1. **关键词命名**：图层名包含业务词（如 `绿化`、`主路`、`商场`），插件会自动匹配预设高度
2. **透明度**：水面、玻璃在 Figma 中设 `Fill Opacity`，插件会生成透明材质
3. **命名放父级**：业务命名放在 `<g>` 父级上，子级形状自动继承父级名
4. **合并形状**：Figma 内尽量 Union 掉同色块，减少面数
5. **曲线精度**：圆弧多的图提高精度（6-8），纯矩形图保持 4 即可

## 六、常见问题

**Q：导入后图层名是乱码？**
Figma 做了编码转义，插件已尽力还原。可在 Figma 侧用英文层名彻底规避。

**Q：拼音防呆导出的文件名也是乱码？**
v0.3.14 内置了拼音引擎，不会出现这个问题。如果遇到异常，检查「拼音防呆」开关是否开启。

**Q：点了"生成色块贴图"报错"未找到材质"？**
先执行「一键挤出」再点色块贴图，按标签页顺序操作。

**Q：一个物体有多个材质球，贴图会对吗？**
完全正确。v0.3.9 起支持按面写 UV，多材质球无需分离物体。

**Q：挤出后某些面朝下长了？**
插件自动做了法线修正，一般不会出现。如果仍有问题，检查 SVG 路径方向。

**Q：GLB 文件太大？**
降低曲线精度（标签 1）、关闭不要的图层（标签 2 眼睛图标），或清理场景中无关物体。

**Q：场景里其他模型被插件改了？**
不会。插件只操作图层列表中注册的地图物体，对其他资产零影响。

## 七、技术要点速览

以下简要说明几个关键实现，方便排查问题：

| 技术点 | 说明 |
|--------|------|
| Figma 编码解码 | `decode_figma_id` 通过 latin1→utf-8 + 十六进制替换 + URL 解码还原中文名 |
| 物理尺寸换算 | `scale_factor = 3543.0 × (ratio_m / ratio_px)`，默认 10px=1m 时乘数 354.3 |
| SVG 层级拉平 | `apply_target_id` 递归把父 `<g>` 的解码名烙印到子形状 `id` 上 |
| 法线修复 | `force_normals_up` 检测 `face.normal.z < -epsilon` 并翻转，epsilon 防浮点抖动 |
| 双轨制图集 | 按材质颜色聚合 → 生成 sRGB 色块贴图 → 分流 Opaque/Transparent 材质 |
| 按面 UV 写入 | 预计算 `material_index → 色块 UV` 映射，`foreach_set` 批量写入 |
| 冗余槽合并 | 替换材质槽后检测重复 → 合并 + 重映射 `poly.material_index` |
| 拼音防呆 | `chinese_to_safe_name` 转拼音 → 导出 → `finally` 块还原原名 |
| 名称去重 | `get_base_name` 剥离 `.001` 后缀；无 ASCII 字符时用 md5 哈希防重名 |
