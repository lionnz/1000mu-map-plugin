# 1000MU 3D MAP 插件 · 技术说明书 v0.3.17

> **适用版本**：v0.3.17 | **兼容 Blender**：4.2.0 – 4.5.x  
> **定位**：设计团队内部工具，Figma 2D 矢量地图 → WebGL 3D 资产全自动化生产管线  
> **安装方式**：ZIP 包（内置 pypinyin 拼音引擎，无需联网安装任何依赖）

## 一、插件简介

这个插件解决一个具体问题：**设计师在 Figma 里画好的 2D 商圈地图，怎么快速变成 3D 沙盘模型？**

传统做法是手动建模——一个一个地块挤出、调颜色、对齐尺寸——非常耗时。本插件把整个流程自动化了。

### 核心工作流

```
Figma 导出 SVG → 导入 Blender（自动换算比例）→ 配置高度和颜色 → 一键挤出 → 材质图集优化 → 导出 GLB
```

## 二、安装

### 首次安装

1. 打开 Blender，**编辑 → 偏好设置 → 插件**
2. 点击右上角 **从磁盘安装**
3. 选择 `1000MU_Map_Plugin_v0317.zip`
4. 勾选启用，点击左下角 **保存用户设置**

5. 回到 3D 视图，按 `N` 键打开侧边栏，点击 **`1000Map`** 标签

### 升级（从 v0.3.16）

1. Blender → 偏好设置 → 插件，找到旧版插件点 **Remove**
2. 点击右上角 **从磁盘安装**，选择 `1000MU_Map_Plugin_v0317.zip`
3. 勾选启用，保存用户设置

### 升级（从 v0.3.13 及更早）

旧版是单 `.py` 文件，v0.3.14 起改为 `.zip` 包，必须彻底清除旧版残留才能正常安装：

1. **关闭 Blender**
2. 打开终端，执行清除命令：
   ```bash
   rm -rf ~/Library/Application\ Support/Blender/4.2/scripts/addons/1000MU_Map_Plugin*
   ```
3. 重新打开 Blender，进入 **编辑 → 偏好设置 → 插件**，检查列表里是否还有残留的 1000MU 条目（如有则点 **Remove** 彻底清理）
4. 再按上方「首次安装」步骤安装新版
5. 新旧版本不能同时启用

## 三、操作指南

插件面板有三个标签页：**导入 → 挤出 → 导出**，按顺序操作。

### 标签 1：导入

#### 区块 A：视图设置

导入大比例尺地图时，Blender 默认 Clip 范围会导致模型破面/消失，提供 6 档一键切换：

| 按钮 | Clip Start | Clip End | 说明 |
|------|-----------|----------|------|
| 默认 | 0.01 m | 1000 m | Blender 默认值 |
| 远点×10 | 0.01 m | 10000 m | 仅拉远 End |
| ×10 | 0.1 m | 10000 m | Start/End 同步拉远 |
| 远点×100 | 0.01 m | 100000 m | 仅拉远 End（超大地图） |
| ×100 | 1 m | 100000 m | Start/End 同步拉远（超大地图） |
| ×1000 | 10 m | 1000000 m | Start/End 同步拉远（超超大地图） |

#### 区块 B：着色方式（v0.3.17 从导出页移入）

快速切换实体显示模式下的光照灯棚与颜色类型，方便检查色块贴图效果：

| 按钮 | 说明 |
|------|------|
| 默认视图着色 | 默认灯棚 + 材质颜色 |
| paint.sl+材质 | paint.sl 灯棚 + 材质颜色 |
| paint.sl+纹理 | paint.sl 灯棚 + 纹理（查看图集效果） |

#### 区块 C：基础导入参数

| 步骤 | 操作 |
|------|------|
| **选 SVG** | 选择 Figma 导出的 `.svg` 文件。插件自动解码 Figma 中文层名 |
| **调比例** | 默认 `10px = 1m`，这是团队标准。可按项目调整 |
| **曲线精度** | 默认 4。圆弧多的图可以调到 6-8（更高=更平滑但面数更多） |
| **点导入** | 自动完成：解码层名 → 换算物理尺寸 → 全局居中 → 提取颜色 → 几何节点填充曲线 → 创建材质 |

> 导入后每个网格已有 `Mat_{图层名}` 标准材质，颜色来自 SVG 解析的线性颜色值。

### 标签 2：挤出（原"配置"页）

#### 区块 A：网格平面优化

**检查0面积的面**：遍历选中网格，面积 < 0.0001 的面视为0面积面，将问题物体**移动至**集合「0面积的面」（从原属集合移除，避免一物体多集合混乱）并报告。

**重新拓扑**：先执行前置优化（按距离合并 0.0001m + 有限融并 0.1°）清理退化网格，再通过 bmesh 提取边界轮廓线 → 转曲线 → 几何节点填充曲线（N-gons 模式）→ 转网格 → 面三角化→转四边面稳定拓扑，消除0面积面，生成干净拓扑。会自动保存恢复材质。

**对所选曲线物体进行网格优化**（v0.3.17 新增）：对选中曲线物体执行 FillCurve 填充 → 转网格 → 面三角化（BEAUTY）→ 三角面转四边面（1°阈值）→ 兜底清理0面积退化面。适用于直接用 Blender 原生导入的 SVG 曲线物体（未经插件导入流程的），生成与「导入SVG并重构空间」一致的稳定拓扑。

#### 区块 B：图层配置列表

**架构变更**：图层列表改为基于当前场景选中的网格物体动态生成。点击「刷新图层列表」按钮，按 base_name 去重生成。

| 控件 | 作用 |
|------|------|
| 色块 | 从 SVG 取的填充色，可手动覆盖 |
| 图层名 | 从 SVG 识别，只读 |
| 🎲 随机开关 | 开启后设一个区间，每个地块随机取高度，产生错落感 |
| 高度(m) | 修改挤出厚度 |

> 导入后自动匹配高度预设表。未匹配的图层随机取 1-10m。

#### 区块 C：一键挤出

配置好后点底部**「一键挤出」**。

**一键挤出做了什么**：
1. 翻转朝下的法线使其朝 +Z 生长
2. 按设定高度 Solidify 挤出
3. Weld 焊接顶点 + Triangulate 三角化（优化拓扑）
4. 生成 Principled BSDF 材质（透明物体自动 BLEND 模式），始终更新颜色属性
5. 清理无用的孤立数据

> 插件只会处理当前选中的地图物体，不会动场景里其他东西。

### 标签 3：导出

#### 区块 A：实用小工具

**从该文件中清理未使用的数据**（v0.3.17 从导入页移入）：递归清理孤立数据块（手动 `bpy.data.X.remove()`，不使用 `orphans_purge` 避免内层 ERROR 报告穿透）。无孤立数据时提示「场景很干净，没有孤立数据块」。

**UVmap 批量清理**：确保每个网格有且仅有一个 UV 层，名称为 `UVMap`。

**自定义法向批量清理**：移除选中网格的自定义拆边法向数据，modal+timer 分批处理避免卡顿。

**按所选面批量设置原点**：编辑模式下，将每个网格物体原点设置到其选中面的中心，旋转对齐完整面坐标系（X=切线, Y=法向×切线, Z=法向）。

**材质球打包、生成色块贴图**：按材质颜色聚合，生成全局色块贴图，自动分流不透明/透明材质，自动合并冗余材质槽。

#### 区块 B：导出预设

| 选项 | 说明 |
|------|------|
| 仅导出可见 | 跳过隐藏的物体 |
| 应用修改器 | 导出时烘焙修改器效果 |
| 导出时静默中文转拼音，防前端报错 | 中文名自动转拼音，防前端报错。导出后**自动还原原名**，不污染 .blend 文件 |

#### 区块 C：导出GLB

点击「导出GLB」按钮，打开 Blender 原生 GLB 导出弹窗，插件勾选项自动同步到弹窗。

**拼音防呆实现**：导出前改名 → 记录原名映射 → 打开原生弹窗 → 计时器监听弹窗关闭 → 延迟2秒 → 还原原名。

> 后台模式（AI 自动化）下计时器不可用，需用同步版拼音防呆，详见 [svg-to-glb-pipeline.md](svg-to-glb-pipeline.md)。

## 四、自定义高度预设

高度预设表可以在 Blender 偏好设置里编辑，不用改代码。

**位置**：编辑 → 偏好设置 → 插件 → 展开「1000MU 3D MAP 插件」

| 操作 | 方法 |
|------|------|
| 添加 | 点 `[+]` |
| 删除 | 选中一行点 `[-]` |
| 编辑 | 直接在表格里改关键词和高度 |
| 恢复默认 | 点「恢复内置默认值」 |

**匹配规则**：图层名**包含**关键词即命中。例如 `主路-辅道` 包含 `主路` → 自动设 20m。多个关键词同时命中时，按表格顺序优先匹配第一个。

### 内置默认值

| 关键词 | 高度 | 关键词 | 高度 |
|--------|------|--------|------|
| 绿化 | 10m | 深色box | 30m |
| 主路 | 20m | 文字 | 0.5m |
| 支路 | 15m | 商场边框 | 60m |
| 中色box | 40m | 商场 | 80m |
| 浅色box | 50m | 水 | 5m |

## 五、团队制图规范

为了插件能精准识别图层，建议在 Figma 中遵守以下约定：

1. **关键词命名**：图层名包含业务词（如 `绿化`、`主路`、`商场`），插件会自动匹配预设高度
2. **透明度**：水面、玻璃在 Figma 中设 `Fill Opacity`，插件会生成透明材质
3. **命名放父级**：业务命名放在 `<g>` 父级上，子级形状自动继承父级名
4. **合并形状**：Figma 内尽量 Union 掉同色块，减少面数
5. **曲线精度**：圆弧多的图提高精度（6-8），纯矩形图保持 4 即可

## 六、常见问题

**Q：导入后图层名是乱码？**
Figma 做了编码转义，插件已尽力还原。可在 Figma 侧用英文层名彻底规避。

**Q：导入后网格没有材质？**
v0.3.15 起导入时直接创建 `Mat_{图层名}` 标准材质。如遇异常，检查 SVG 文件是否包含有效的 fill 属性。

**Q：拼音防呆导出的文件名也是乱码？**
内置了拼音引擎，不会出现这个问题。如果遇到异常，检查「拼音防呆」开关是否开启。

**Q：点了"生成色块贴图"报错"未找到材质"？**
先执行「一键挤出」再点色块贴图，按标签页顺序操作。

**Q：一个物体有多个材质球，贴图会对吗？**
完全正确。v0.3.9 起支持按面写 UV，多材质球无需分离物体。

**Q：挤出后某些面朝下长了？**
插件自动做了法线修正，一般不会出现。如果仍有问题，检查 SVG 路径方向。

**Q：GLB 文件太大？**
降低曲线精度（标签 1）、关闭不要的图层，或清理场景中无关物体。

**Q：场景里其他模型被插件改了？**
不会。插件只操作当前选中的地图物体，对其他资产零影响。

**Q：重新拓扑后材质丢失？**
v0.3.15 已修复此问题，重新拓扑会自动保存恢复材质。

**Q：清理未使用数据只显示红叉不显示文案？**
v0.3.17 已修复此问题。根因是 UI 中使用了无效图标 `ROCKET` 导致 `draw()` 异常，污染了报告系统。已改为有效图标。

**Q：后台模式（AI 自动化）导出怎么用？**
详见 [svg-to-glb-pipeline.md](svg-to-glb-pipeline.md)，含完整流程脚本模板。

## 七、技术要点速览

以下简要说明几个关键实现，方便排查问题：

| 技术点 | 说明 |
|--------|------|
| Figma 编码解码 | `decode_figma_id` 通过 latin1→utf-8 + 十六进制替换 + URL 解码还原中文名 |
| 物理尺寸换算 | `scale_factor = 3543.0 × (ratio_m / ratio_px)`，默认 10px=1m 时乘数 354.3 |
| SVG 层级拉平 | `apply_target_id` 递归把父 `<g>` 的解码名烙印到子形状 `id` 上 |
| 导入网格优化 | 几何节点 FillCurve（N-gons 模式）替代传统 2D 填充，正确识别镂空/孔洞；烘焙后执行面三角化→转四边面稳定拓扑 |
| 法线修复 | `force_normals_up` 检测 `face.normal.z < -epsilon` 并翻转，epsilon 防浮点抖动 |
| 重新拓扑 | 前置优化（按距离合并+有限融并）→ bmesh 删面+删非边界边提取轮廓线 → 曲线 → 几何节点填充 → 网格 → 面三角化→转四边面稳定拓扑，含材质保存恢复 |
| 曲线网格优化 | 对曲线物体直接创建 FillCurve 几何节点 → 转网格 → 面三角化→转四边面 → 兜底清理0面积面，与导入流程算法一致 |
| 色块图集 | 按材质颜色聚合 → 生成色块贴图 → 分流不透明/透明材质 |
| 按面 UV 写入 | 预计算 `material_index → 色块 UV` 映射，`foreach_set` 批量写入 |
| 冗余槽合并 | 替换材质槽后检测重复 → 合并 + 重映射 `poly.material_index` |
| 拼音防呆 | `chinese_to_safe_name` 转拼音 → 导出 → `bpy.app.timers` 计时器还原原名 |
| 按所选面设原点 | 完整面坐标系对齐（X=切线, Y=法向×切线, Z=法向），切线取面第一条边方向 |
| 名称去重 | `get_base_name` 剥离 `.001` 后缀；无 ASCII 字符时用 md5 哈希防重名 |
| 孤立数据清理 | 手动 `bpy.data.X.remove()` 递归移除 users==0 的数据块，避免 `orphans_purge` 的内层 ERROR 报告穿透 |

## 八、源码模块结构

v0.3.17 将 `build_atlas` 和 `purge_scene` 移入 `repair_tools.py`，删除 `scene_tools.py`，代码分块与 UI 面板对应：

```
1000MU_Map_Plugin/
├── __init__.py          # bl_info、注册入口
├── constants.py         # 常量（INVALID_LAYER_NAMES, BUILTIN_HEIGHT_PRESETS 等）
├── utils.py             # 工具函数（拼音、SVG解析、法线修复等）
├── props.py             # PropertyGroup 定义
├── preferences.py       # 偏好设置（高度预设表）
├── ui.py                # N 面板 UI 绘制
└── operators/
    ├── __init__.py      # 操作符注册
    ├── pipeline.py      # 核心流程（导入、刷新图层、挤出、导出）
    ├── mesh_tools.py    # 网格工具（0面积检查、重新拓扑、曲线网格优化）
    ├── repair_tools.py  # 修复与清理工具（清理孤立数据、UV清理、法向清理、设原点、色块贴图）
    └── view_tools.py    # 视图工具（视图裁剪、着色方式）
```

**模块与 UI 面板对应关系：**

| 模块文件 | UI 位置 |
|---------|---------|
| pipeline.py | 核心流程（导入、挤出、导出） |
| view_tools.py | 导入-视图设置（裁剪预设 + 着色方式） |
| mesh_tools.py | 挤出-网格平面优化 |
| repair_tools.py | 导出-实用小工具（清理孤立数据、UV清理、法向清理、设原点、色块贴图） |
| presets.py | 偏好设置（高度预设表） |

## 九、兼容性

| Blender 版本 | 状态 | 说明 |
|-------------|------|------|
| 4.2.x | ✅ 完全兼容 | 已通过完整流程测试 |
| 4.5.x | ✅ 完全兼容 | 已通过完整流程测试 |

### Blender 4.x API 注意事项

- 几何节点组类型用 `'GeometryNodeTree'`（非 `'GEOMETRY'`）
- 几何节点 socket 用 `ng.interface.new_socket()`（非 `ng.inputs.new()`）
- `BMFace` 无 `calc_tangent()`，用面第一条边方向作为切线
- `mat.diffuse_color` 与 `bsdf.inputs['Base Color']` 都是线性值，不需要 sRGB 转换
- UI 图标必须是 Blender 4.x 有效枚举值（如 `ROCKET` 在 4.2 中不存在，会导致 `draw()` 异常并污染报告系统）

## 十、调试经验

遇到「功能正常但提示异常」的 BUG 时，推荐用**终端启动 Blender** 查看系统日志：

```bash
/Applications/Blender\ 4.2\ LTS.app/Contents/MacOS/Blender
```

这样 Blender 的 `print()` 输出和 `Traceback` 会直接显示在终端，便于定位 UI `draw()` 异常等后台模式无法复现的问题。详见 [blender-plugin-debug-guide.md](blender-plugin-debug-guide.md)。
